# Codex 操作规范

本仓库用于学习多智能体强化学习、MPE 环境、PPO、QMIX 和 MADDPG。后续 Codex 协作时请遵守以下规范。

## 修改前

- 每次修改前先说明计划，简要写清楚准备改哪些文件、为什么改。
- 先查看现有目录和文件内容，再决定修改方式。
- 优先保持项目结构清晰，不提前引入复杂抽象。
- 如果任务涉及实验结果、日志或模型文件，先确认路径和影响范围。

## 修改中

- 保持改动范围尽量小，只处理当前任务需要的内容。
- 不要写复杂算法占位代码；学习仓库应按阶段逐步实现。
- 新增代码应尽量清晰、可读，并符合当前目录约定。
- 不要随意移动、重命名或删除已有文件。

## 修改后

- 每次修改后说明改了什么。
- 如果运行了检查、测试或示例命令，说明结果。
- 如果没有运行检查，也要明确说明。
- 对后续建议保持简短，避免一次性引入太多方向。

## 实验输出保护

- 不要删除已有实验结果。
- 不要改动 `outputs/` 里的历史记录。
- 不要覆盖已有 checkpoint、TensorBoard 日志、图表或实验日志。
- 如需生成新的实验输出，应使用新的时间戳目录或新的实验名。
- 如需清理输出文件，必须先得到用户明确确认。

## Git 与依赖

- 不要提交大型模型文件、缓存文件或虚拟环境。
- 不要自动升级大量依赖，除非用户明确要求。
- 修改 `requirements.txt` 时说明新增依赖的用途。
